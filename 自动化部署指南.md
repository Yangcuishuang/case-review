# 自动化部署指南

## 方案一：一键推送脚本 ✅ 推荐

### 前置条件

需要先配置 GitHub 认证（二选一）：

#### 方式 A：HTTPS + 个人访问令牌（推荐新手）

1. **创建 GitHub Token**
   - 访问：https://github.com/settings/tokens
   - 点击 **"Generate new token (classic)"**
   - 勾选 **"repo"** 权限
   - 点击 **"Generate token"**
   - **复制 Token**（只显示一次，务必保存！）

2. **配置 Git**
   ```bash
   git config --global credential.helper store
   ```

3. **首次推送时输入认证**
   - Username: 你的 GitHub 用户名
   - Password: 粘贴刚才的 Token（不是密码）

#### 方式 B：SSH 密钥（推荐长期使用）

```bash
# 生成 SSH 密钥
ssh-keygen -t ed25519 -C "你的邮箱@example.com"

# 查看公钥
cat ~/.ssh/id_ed25519.pub

# 复制公钥内容，添加到 GitHub
# 访问：https://github.com/settings/keys → New SSH key

# 修改远程地址为 SSH
cd /Users/yangyangliang/CodeBuddy/20260324201828
git remote set-url origin git@github.com:Yangcuishuang/case-review.git
```

## 方案二：使用 GitHub API（高级）

如果你希望完全自动化（不需要本地 Git），可以使用 GitHub API：

### 创建 API 更新脚本

```python
# update_github.py
import requests
import base64

# 配置
TOKEN = "你的GitHub Token"
REPO = "Yangcuishuang/case-review"
FILE_PATH = "index.html"

def update_file(content, message="更新网页"):
    # 获取当前文件 SHA
    url = f"https://api.github.com/repos/{REPO}/contents/{FILE_PATH}"
    headers = {"Authorization": f"token {TOKEN}"}
    
    response = requests.get(url, headers=headers)
    sha = response.json()["sha"]
    
    # 更新文件
    data = {
        "message": message,
        "content": base64.b64encode(content.encode()).decode(),
        "sha": sha
    }
    
    requests.put(url, headers=headers, json=data)
    print("✅ 更新成功！")

# 使用
with open("index.html", "r", encoding="utf-8") as f:
    update_file(f.read(), "更新案例数据")
```
